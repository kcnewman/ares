# ARES (Automated Residential Estimation System)

ARES is an end-to-end MLOps project for estimating rental price ranges for residential properties in Greater Accra, Ghana. It includes a full training pipeline, model tracking, batch inference, a FastAPI service, and a Streamlit frontend.

## Tech Stack

- Python 3.12
- CatBoostRegressor
- FastAPI + Uvicorn
- Streamlit
- MLflow
- `uv` for environment and dependency management

## End-to-End Flow

```
Raw CSV -> Data Validation -> Data Split -> Data Processing -> Feature Engineering
-> Model Training -> Model Evaluation + MLflow -> Inference (CLI/API/UI)
```

## Quick Start

### 1. Prerequisites

- Python `3.12.x`
- `uv` installed
- Raw dataset at `artifacts/data/raw.csv` (default path in `config/config.yaml`)

### 2. Install Dependencies

```bash
uv sync --dev
```

### 3. Optional Environment Variables

```bash
export GOOGLE_MAPS_KEY="your-key"                  # for geocoding fallback
export MLFLOW_TRACKING_URI="sqlite:///experiments/mlflow.db"
```

### 4. Run Full Training Pipeline

```bash
uv run python main.py
```

This executes all stages in order and packages non-CSV artifacts into `artifacts.zip`.

### 5. Run API Locally

```bash
uv run uvicorn ares.api.main:app --host 127.0.0.1 --port 8000 --reload
```

### 6. Run Streamlit UI Locally

```bash
BACKEND_URL="http://127.0.0.1:8000" uv run streamlit run app.py
```

### 7. Run Batch Inference

```bash
uv run python src/ares/pipeline/inference.py \
  --input artifacts/data/raw.csv \
  --model artifacts/model_trainer/model.joblib
```

### 8. Run Tests

```bash
uv run pytest -q
```

## Core Artifacts

- `artifacts/data_validation/status.txt`: validation gate output.
- `artifacts/data_split/*.csv`: train/eval split.
- `artifacts/data_processing/*.csv`: cleaned datasets.
- `artifacts/feature_engineering/*.csv|*.json`: features and locality statistics.
- `artifacts/model_trainer/model.joblib`: trained CatBoost model.
- `artifacts/model_evaluation/metrics.json`: evaluation metrics.
- `artifacts/inference/predictions.csv`: batch prediction output.
- `artifacts.zip`: deployment package generated by `main.py`.

## Deployment

Deployment is automated through GitHub Actions (`.github/workflows/deploy.yml`) and targets Google Cloud Run.

- Trigger: push to `main` with changes to `artifacts.zip`.
- Build: Docker image pushed to Artifact Registry.
- Release: Cloud Run service `ares-service` in `us-central1`.

Full runbook: [`docs/OPERATIONS.md`](docs/OPERATIONS.md)
